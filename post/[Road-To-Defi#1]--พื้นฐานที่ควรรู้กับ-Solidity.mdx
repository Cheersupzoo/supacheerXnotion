ก่อนเข้าสู่เนื้อหา อยากเตือน⚠️ก่อนว่า บทความนี้เป็นการเล่าสิ่งที่ผู้เขียนพบเจอระหว่างการศึกษา Solidity โดยใช้มุมมองความเป็น Full Stack dev ที่ผู้เขียนชำนาญกว่ามานำเสนอ ⚠️⚠️ ณ ขณะนี้ผู้เขียนเพิ่งเริ่มศึกษาภาษา Solidity แต่อยากนำเสนอมุมมองที่น่าสนใจ ชี้ให้เห็นความแตกต่างจากภาษาอื่นๆ นำมาสรุปให้ฟังกันครับ

หลายๆคน คงจะคล้ายกับผู้เขียนที่สนใจศึกษาภาษา Solidity เพราะเห็นการเติบโตของ Defi (Decentralize Finance) ที่โตเอาอย่างมากในยุคปัจจุบัน แล้วอยากกระโดดเข้ามาเป็นส่วนหนึ่งของมัน แล้วก็จะค้นพบว่า Defi เหล่านี้ถูกสร้างขึ้นมาด้วย Smart Contract ที่ทำงานบน Block Chain เจ้าดังๆ อย่างเช่น Ethereum, Binance Smart Chain หรือ Bitkub Chain ที่เพิ่งเกิดใหม่ โดยตัว Smart Contract ก็จะถูกพัฒนาด้วยภาษา Solidity ซึ่งถูกสร้างเพื่อทำงานบน EVM(Ethereum Virtual Machine) 

ดังนั้น Solidity ก็แค่ภาษาใหม่ ที่มี Syntax เปลี่ยนไปจากภาษาอื่น — ซึ่งก็อาจจะใช่ แต่สิ่งที่ทำให้ Solidity แตกต่างจากภาษาอื่น ก็เพราะภาษานี้ไม่ได้ถูกสร้างเพื่อพัฒนา Backend แบบเดิมๆ Solidity ถูกสร้างเพื่อใช้เป็น Backend+Database สำหรับ Block Chain แต่ก็ได้อานิสงส์ ของภาษาสมัยใหม่ๆมาบ้าง โครงสร้างภาษาก็จะคล้ายๆกับ JavaScript, C++, Python มาผสมกัน มีโครงสร้างแบบ Object Oriented programming (OOP)

ในบทความนี้ เราจะยังไม่ได้ลงลึกกับโค๊ด แต่จะไปโฟกัสที่แนวคิด และโครงสร้างพื้นฐานของ EVM ที่จะส่งผลกับการพัฒนา Solidity

ก่อนอื่น เราต้องเปิดมุมมองใหม่ ว่าเราไม่ได้พัฒนา Backend แบบเดิมๆอีกต่อไป โค๊ดเราไม่ได้ถูกนำไปรันในสภาพแวดล้อมปิด ที่เราจะสามารถซ่อนโค๊ดการทำงานต่างๆของเราได้ แต่โค๊ดของเราจะมีความเปิดเผย บนโลกของ Block Chain ใครก็สามารถเข้ามาดู ซึ่งก็เป็นดาบสองคม เพราะว่า คนที่มาใช้บริการของเราสามารถตรวจสอบความถูกต้องได้เอง แต่ผู้ที่ไม่หวังดีก็สามารถทำได้เช่นกัน แล้วสามารถมองหาช่องโหว่ได้ง่ายขึ้น ดังนั้นจึงจำเป็นที่ต้องมีความระมัดระวังเป็นอย่างยิ่ง โดยเฉพาะเมื่อ App (dApp) ส่วนใหญ่เกี่ยวข้องกับการเงิน การทำผิดพลาดแต่ละครั้ง จะสามารถสร้างมูลค่าความเสียหายได้มาก6
